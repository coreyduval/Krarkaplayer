OPEN QUESTIONS FOR THE USER (judgment calls — NOT auto-fixed)
====================================================================
No CLEAR-CUT, single-line illegal move was found that directly contradicts
Scryfall text in a mechanically-fixable way. All concerns below are documented,
UNIFORM solitaire abstractions. Per the fix policy ("solitaire model may abstract
opponents"), these are judgment calls left for the user. NO code was edited.

Q1. CHROME MOX — no imprint tracked.
  Scryfall (Chrome Mox): "Imprint — When this artifact enters, you may exile a
  nonartifact, nonland card from your hand. {T}: Add one mana of any of the exiled
  card's colors."  Engine (tables.rs:33) models Chrome Mox as Tap -> {*} (any color),
  UNCONDITIONALLY, and never (a) pays the card-exile cost nor (b) restricts the color
  to the imprinted card's colors. Impact: Chrome Mox is treated as a free, colorless,
  any-color rock; in reality it costs a card and makes NOTHING if you imprint nothing.
  Seen live: seed 29 luck 4 (Chrome Mox -> {*} on T1 with no card exiled).
  -> Possible mild win-inflation. JUDGMENT CALL: usually you have a colored card to
     imprint, so {*} is realistic, but the card-loss is never charged.

Q2. MOX AMBER / COMMAND TOWER / others -> {*} ignore color-identity/legend conditions.
  Scryfall (Mox Amber): "{T}: Add one mana of any color among legendary creatures and
  planeswalkers you control." Command Tower: any color in commander's color identity
  (= U/R here, not all 5). Engine models both as {*} (any of 5 colors). For THIS deck
  the practical colors needed are only U/R/C, and a legend is essentially always out,
  so {*} vs the true restricted set is behaviorally equivalent here. Flagged for
  completeness; low/zero impact.

Q3. LOOPABLE COUNTERS assume an opponent spell is always on the stack.
  loops::MAGECRAFT_FUEL = {Flusterstorm, Deflecting Swat, Fierce Guardianship, Pact of
  Negation, An Offer You Can't Refuse, Mogg Salvage, Cyclonic Rift} are cast in-loop
  purely for magecraft/storm value. Counters REQUIRE a legal target:
   - An Offer (Scryfall ruling): "If the target is no longer legal as An Offer resolves,
     no Treasure tokens are created." Engine credits An Offer 2 treasures/cast
     (loops.rs spell_generic_per_resolution) — only valid if a noncreature spell is
     actually on the stack to counter each cast. The code comment posits "counter your
     own already-triggered spell -> you're controller -> 2 treasures," which is a real
     but FRAGILE assumption (you must have a second castable spell on the stack first).
   - Cyclonic Rift / Flusterstorm / Fierce Guardianship likewise need a target.
  The engine explicitly assumes opponents always present a legal counter-target
  (cards.rs NO_SOLITAIRE_TARGET comment). This is the standard solitaire abstraction;
  it can over-credit loop fuel/treasures in lines with no real opponent spell.
  -> JUDGMENT CALL: acceptable as a solver abstraction, but it can inflate go-off
     probability for lines that lean on counters-as-fuel. User to decide if these
     should be excluded from the loop-fuel set for a stricter model.

Q4. FRANTIC SEARCH / SNAP "untap up to N lands" credited as +N wildcard mana.
  resolver.rs untap_mana adds untap_cap(card) * resolutions of {*}, capped by land
  count on board. Real cards UNTAP lands (re-tappable for their own colors), so {*}
  is a slight over-generosity (an untapped Mountain makes R, not any color). Low
  impact for a U/R deck; flagged for completeness.

CONCLUSION: Engine is mechanically sound on hard-illegality axes (mana never negative,
treasures spent <= made, legend rule honored via Sakashima immunity, command tax,
zone/enabler gating, payoff accessibility). No clear-cut auto-fix applied. The items
above are uniform abstractions for the user to rule on, not bugs.
